# glfw

이제 opengl context를 만들고 window를 만들어야 한다.

그 동작은 os마다 다르고 opengl은 그런 작업을 안하고싶어한다. opengl은 창이 뭔지, 키보드가 뭔지, 화면이 뭔지 모른다. opengl은 프레임버퍼에 픽셀 그리는 것밖에 모른다.

그래서 그런걸 해주는 opengl을 타겟으로 한 라이브러리가 있다. 그중하나가 glfw다.

glfw는 opengl context만들기, window 정의하기, 유저 인풋 받기 등을 처리해준다.

튜토리얼에서는 glfw의소스코드를 다운받고 cmake도 다운받았다. 나도 cmake를 다운받았다. 잘 모르겠지만 아마 cmake가 glfw 소스코드를 컴파일하는용도라는 것 같다. 근데 나는 glfw를 터미널에서 brew로 다운받았다. 그래서 brew가 컴파일도 다 해준건지? cmake는 깔필요가 없었다는 듯하다. 그래도 c 쓸때 이래저래 cmake가 필요하고 용량도 작으니 굳이 삭제할필요 없다고 챗지피티가 그래서 내버려 뒀다.

# glad

opengl driver가 너무 많아서 함수들이 어디있는지 컴파일타임에 모른다 -> 런타임에 함수의 위치를 가져와서 function pointer에 담아두는 것은 개발자의 몫이 된다

하지만 그것 또한 os에 따라 방법이 다르고 너무 복잡하다.

그래서 그걸 해주는 라이브러리가 있다. glad이다. glad 웹사이트에 가서 opengl 버전 뭐쓸지 정하면 그버전의 함수를 다가져오는 glad 파일을 다운받을 수 있다.
